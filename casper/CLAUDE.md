## Explanation
This project is a customized Casper theme (one of the default Ghost themes) for Ghost blogging. The goal is to adjust it to personal needs and add aesthetic visual elements with a retro/pixel art style.

## Theme Verification
Before uploading the theme, it must be verified via gscan.ghost.org.
To verify, create a zip file by running:
```
npm run zip
```
This invokes `gulp zip` (defined in `gulpfile.js`), which builds CSS + JS then zips everything into `dist/casper-modified.zip`.
Run `npm install -f` first **only if the `node_modules` directory is not already present**.

## Design System

### Color Palette
CSS custom properties injected by `default.hbs` from Ghost Admin > Design > Custom settings, with fallbacks in `custom.css`:

| Variable       | Landscape value | Portrait value  | Purpose                        |
|----------------|-----------------|-----------------|--------------------------------|
| `--theme`      | `#141c26`       | `#1b4065`       | Page background, nav bg        |
| `--theme-lt`   | `#141c26`       | `#1b4065`       | Light variant of theme bg      |
| `--text`       | `#dfdaf0`       | `#000000`       | Body text color                |
| `--hero-bg`    | `#FEF3E2`       | —               | Hero card background           |
| `--orange`     | `var(--ghost-accent-color, #F5A623)` | — | CTA buttons, accents   |
| `--orange-dk`  | `var(--ghost-accent-color, #E8950F)` | — | Hover state of orange  |

Portrait mode overrides `--theme`, `--text`, `--text-rgb` via `@media (orientation: portrait)` in `custom.css`.

### Typography
- **Pixel font**: `Press Start 2P` (Google Fonts) — applied to nav, hero card, post titles, headings, footer. Loaded in `default.hbs` `<head>`.
- **Body font**: Default Ghost/Casper font stack.
- All images use `image-rendering: pixelated` for a retro look.

### Scrollbar
Custom retro Win95-style scrollbar defined in `custom.css` via `::-webkit-scrollbar` and `scrollbar-color` (Firefox). In portrait mode the scrollbar track becomes transparent and the thumb becomes `#cd8d38`.

## About the Theme and Layout

### Two-orientation layout
The theme has two distinct visual layouts switched entirely via CSS `@media (orientation: portrait/landscape)`. No JS is needed to switch; the browser handles it automatically.

#### Landscape mode (default)
- Dark navy background (`#141c26`), light lavender text (`#dfdaf0`).
- Hero section: full-viewport video background (`wall.webm`), split into left text area (50%) and right transparent visual area.
- Navigation bar: dark theme color, search button visible.
- Post card images: grayscale + opacity overlay.
- Scrollbar: retro Win95 raised-3D style.

#### Portrait mode
- Background color changes to `#1b4065` (deep blue), text to `#000000` and `#bb7888` (mauve-pink).
- Hero section: switches to vertical video (`wall_vertical.webm`).
- Navigation bar: `#bb7888` background.
- Post pages: page background becomes transparent, revealing a fixed full-screen video (`wall_vertical.webm`) filtered to grayscale with a `rgba(27,64,101,0.3)` overlay — defined in `default.hbs` and shown only via the portrait media query.
- Scrollbar: transparent track, `#cd8d38` amber thumb.

### Hero Section (`index.hbs` + `custom.css`)
The hero occupies 100vh and consists of:
- **Background video**: a `<video>` tag absolutely positioned behind all content (`z-index: 0`), filling the viewport. Two videos are in the DOM; CSS hides/shows the correct one per orientation.
  - Landscape: `.hero-bg-landscape` — `wall.webm`, `object-fit: fill`
  - Portrait: `.hero-bg-portrait` — `wall_vertical.webm`, `object-fit: fill`
  - On mobile (`max-width: 767px`): `object-fit: cover` is applied to both.
- **Hero card** (`.hero-card`): transparent overlay split into:
  - Left `.hero-text-area` (50%): site title, description with blinking cursor animation, CTA button ("More Info").
  - Right `.hero-visual-area`: empty transparent spacer — the background video is visible through it.
- **Decorative dots** (`.hero-dots`): hidden on mobile (`max-width: 767px`), currently hidden at all sizes (`display: none` in CSS — present in markup for potential future use).

#### Known issue: video autoplay on mobile reload (FIXED)
On mobile browsers, the `autoplay` attribute is sometimes ignored after a page reload because the browser restores the page from **BFCache** (back/forward cache) with the video in a paused state. The fix lives in `default.hbs` as an inline `<script>` after the jQuery block. It explicitly calls `.play()` on all `video[autoplay]` elements on three events:
- `pageshow` with `event.persisted === true` — BFCache restore (reload/back navigation)
- `visibilitychange` (when `document.hidden` becomes false) — tab switching
- `load` — initial load fallback

The selector `video[autoplay]` covers all three video elements (landscape hero, portrait hero, post-page background), so both orientation modes are handled.

### Post Page (`post.hbs` + `default.hbs`)
- `default.hbs` always renders a `<video class="post-bg-video">` (portrait background) and a `.post-bg-overlay` div, but they are `display: none` by default.
- In portrait mode, `body.post-template .post-bg-video` becomes `display: block; position: fixed; inset: 0` — a full-screen fixed video background with `grayscale(100%)` filter and a semi-transparent overlay.
- Feature images on post pages are desaturated (`grayscale(100%)`) and dimmed (`opacity: 0.3`) with a blue tint overlay.

## Build Pipeline (`gulpfile.js`)
- `gulp css` — PostCSS: `postcss-easy-import` → `postcss-color-mod-function` → `autoprefixer` → `cssnano`. Source: `assets/css/*.css`. Output: `assets/built/`.
- `gulp js` — Concatenates `assets/js/lib/*.js` then `assets/js/*.js` into `casper.js`, then uglifies. Output: `assets/built/`.
- `gulp zip` — Runs `css` + `js` then zips the entire theme (excluding `node_modules`, `dist`, `yarn.lock`, `gulpfile.js`) into `dist/casper-modified.zip`.
- `gulp` (default) — Runs `build` then starts livereload watcher.

**Key rule**: always edit source files in `assets/css/` and `assets/js/`. Never edit `assets/built/` directly — it is regenerated on every build.
